Callbacks são funções passadas como argumentos para outras funções para serem chamadas posteriormente, quando uma operação assíncrona for concluída ou um evento ocorrer. Elas permitem controlar o fluxo de execução do código e definir ações personalizadas de acordo com eventos específicos.

Uma operação assíncrona é uma operação que não bloqueia a execução subsequente do código. Em vez disso, a operação é iniciada e o código continua sua execução sem esperar que a operação seja concluída. Em contraste, uma operação síncrona é uma operação que bloqueia a execução subsequente do código até que seja concluída. Isso significa que o código espera que a operação síncrona termine antes de prosseguir.

O conceito de callbacks está relacionado à programação assíncrona, onde algumas operações podem levar tempo para serem concluídas. Em vez de bloquear a execução do código até que a operação seja finalizada, as funções de callback permitem que o código continue sendo executado e especifiquem o que deve acontecer quando a operação for concluída.
